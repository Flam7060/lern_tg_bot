# lern_tg_bot

Телеграм-бот для обучения программированию. Цель проекта — развивать навыки
разработки: реализовывать разный функционал и пробовать разные методики
(например, TDD).

## Быстрый старт

Сначала настройка — скопировать пример окружения и заполнить данные
(токены, параметры):

```bash
cp .env.example .env
```

Запуск есть в двух вариантах. Приоритетный — в контейнере:

```bash
docker compose up -d
```

Локальный пока тоже поддерживается:

```bash
uv sync
uv run python -m app
```

## Разработка

Соглашения:

- **Стиль кода** — PEP 8, проверяется и форматируется ruff (плюс правила
  импортов, типичных багов и безопасности).
- **Коммиты** — по [Conventional Commits](https://www.conventionalcommits.org)
- **Тесты** — в `tests/`, зеркалят структуру `app/`.

Все действия — через `make` (`make help` — полный список). Перед пушем достаточно
`make ci`: гоняет то же, что GitHub Actions (ruff + pytest), поэтому «локально
прошло, в CI упало» не бывает.

CI состоит из двух workflow: `ci.yml` (ruff + pytest, основной гейт) и
`mega-linter.yml` (Dockerfile/YAML/секреты; Python отдан ruff и отключён).
